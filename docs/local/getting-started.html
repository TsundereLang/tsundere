Getting Started

Introduction

This section teaches the real Tsundere workflow as it exists in this repository: install the CLI, create a project, understand the files it writes, run development mode, build production output, troubleshoot setup, update the tool, and use the local docs.

What This Section Covers

Tsundere is not just a syntax experiment. The CLI in `src/cli.ts` creates starter projects, installs a local Discord runtime, wraps npm installs through the package optimizer, compiles `.yuri` files, runs a generated Node runtime, opens local HTML docs, checks updates from GitHub releases, and performs doctor checks for Node, npm, paths, permissions, and the package store.

The pages in this section stay close to that code. When a command is experimental, the docs say so. When a file is generated by the CLI, the page names the exact path. When setup differs between Windows and Linux, the platform-specific page explains the difference instead of pretending there is one universal install path.

By the end of the section, you should understand the full local loop: install the CLI, create a Discord starter, run npm through Tsundere, put secrets in `.env`, edit `.yuri` files, let dev mode rebuild, run a production-style build, and diagnose the common points where Node, npm, Discord intents, generated runtime files, or the package optimizer can fail.

The Shortest Real Path

Once the CLI is installed, the actual happy path is short. These commands are implemented in the CLI today:

tsundere create my-bot --template discord
cd my-bot
tsundere install
tsundere dev

`tsundere create` writes the starter files. `tsundere install` runs npm through Tsundere's optimizer and repairs the local `@tsundere/discord` runtime when needed. `tsundere dev` compiles `.yuri`, writes runtime output into `.tsundere/runtime-build`, launches Node, and watches for changes.

That shortest path assumes the CLI is already installed, Node and npm are visible in the terminal, and your Discord bot token is added before the bot logs in. If any step fails, do not skip ahead. Run `tsundere doctor`, check the generated files named in the error, and compare your project against the structure page.

How To Use This Section

If you are brand new, read the pages in order. If you already know Node and Discord.js, skim Installation, First Project, and Project Structure first, then spend more time on `.yuri Files` and Development Mode. Those pages explain the Tsundere-specific parts that are easiest to misread if you assume this is just a thin Discord.js wrapper.

If you are debugging a failed install, jump straight to Installation Troubleshooting, then return to Windows Setup or Linux Setup depending on the machine. If you are preparing a release or updating the CLI, read Updating Tsundere and Reading Release Notes before running automatic updater commands.

The docs are local HTML files, not a hosted documentation app. That is intentional: they can ship with the CLI, work offline after install, and be checked by `npm run docs:check` in this repository.

Reality Check

Tsundere is pre-alpha. The compiler is useful, but parts of `.yuri` lowering still rely on targeted transforms rather than a complete AST parser. The distributed runtime has metrics, stress tests, local IPC/cache/task primitives, and planning, but full multi-machine orchestration is not finished. These docs are written to that reality so beginners do not get trapped by imaginary features.
