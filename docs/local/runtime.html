Runtime

The Tsundere runtime is the executable Node-compatible layer that runs generated JavaScript, resolves local runtime packages, and gives developers `tsundere start` instead of manual Node commands.

Overview

Runtime output is written to `.tsundere/runtime-build`. This keeps production startup separate from source output and lets Tsundere apply runtime-specific lowering, module path fixes, and launcher behavior.

Why It Exists

Early Tsundere builds required users to call `node build/main.ts`, which was brittle and confusing. The runtime exists so `tsundere build` and `tsundere start` become the stable mental model.

When To Use It

  • Use `tsundere start` after `tsundere build`.
  • Use `tsundere dev` for watch mode, rebuild, and restart.
  • Use `tsundere runtime install` when `@tsundere/discord` cannot be resolved.
  • Use runtime logs to confirm command sync and gateway startup.

Runtime Package Resolution

{
  "dependencies": {
    "@tsundere/discord": "file:.tsundere/runtime/discord"
  }
}

The local package is copied into the project by the CLI so generated code can import `@tsundere/discord` normally.

Runtime Commands

tsundere runtime install
tsundere build
tsundere start
tsundere dev

Advanced Example

# Production startup shape
tsundere doctor
tsundere runtime install
tsundere install
tsundere build
tsundere start

This sequence verifies the environment, repairs local runtime dependencies, installs packages, builds source, and starts the generated app.

Common Mistakes

  • Do not start `build/main.ts` directly.
  • Do not delete `.tsundere/runtime/discord` unless you plan to reinstall the runtime.
  • Do not commit runtime cache directories unless a specific generated artifact is meant to be versioned.
  • Do not assume the bot is fully connected just because the process started; check gateway readiness and Discord status.

Performance Notes

Production startup should avoid unnecessary rebuilds. Build once, start many times, and keep runtime output small. Watch mode is for development and should not be used as a production supervisor.
