Templates

Templates are the files created by `tsundere create`. They are intentionally small: a Tsundere starter should teach the runtime shape without hiding a full framework behind generated code.

Important Implementation Detail

The current CLI does not copy the `templates/` folders directly. `tsundere create` uses `templateFiles()` and `templateMain()` in `src/cli.ts` to generate files in memory. The `templates/` directory still contains reference starter files, but the command behavior comes from `src/cli.ts` today. When the docs and disk templates disagree, check the CLI source first.

Create Syntax

The first non-flag argument is the folder name. The `--template` flag chooses the starter type. If the folder already exists, the command fails instead of overwriting it.

tsundere create my-bot --template discord
tsundere create api --template rest
tsundere create socket --template websocket
tsundere create worker --template microservice
tsundere create tool --template cli
tsundere create scratch --template empty

After writing files, the CLI tries to install the bundled Discord runtime into `.tsundere/runtime/discord` when the built CLI contains `dist/discord`. If you run `create` from an unbuilt source checkout, that local runtime copy may be skipped until you run `npm run build`, `tsundere runtime install`, or `tsundere install` from a properly built CLI.

Files Every Generated Project Gets

package.json
tsundere.config.json
.env.example
.gitignore
src/main.yuri

`package.json` is an ES module package with scripts for `dev`, `build`, `start`, `lint`, `format`, and `test`. It also includes `@tsundere/discord` as `file:.tsundere/runtime/discord`, even for non-Discord templates in the current generator. That keeps the local runtime available but also means non-Discord starters still carry the Discord dependency until the generator is split more finely.

`tsundere.config.json` defaults to `source: "src"`, `outDir: "build"`, `target: "typescript"`, `strict: true`, `sourceMaps: true`, and command discovery under `./src/commands` with an admin group override. `.gitignore` excludes `node_modules`, `build`, `.tsundere`, and `.env`. `.env.example` starts with `DISCORD_TOKEN=replace-me`.

Discord

Creates a Discord client with `Client`, `Intents`, `Slash`, and `Embed`, registers `/ping`, replies with an embed, and calls `client.login()`. It also adds `src/commands/ping.yuri` for command discovery metadata.

REST

Creates an Express app with a `/health` route and listens on `env.PORT ?? 3000`. You still need installable npm dependencies for Express in the project before it runs.

WebSocket

Creates a `ws` `WebSocketServer` on `env.PORT ?? 3000` and sends an online message to new connections. Like REST, the package dependency must be installed in the project.

Microservice

Creates `handleJob(input: { id: string })` and returns `{ id, ok: true }`. It is a small module starter, not a queue runner or worker framework.

CLI

Reads `process.argv.slice(2)`, defaults to `help`, and prints a help message. It demonstrates `.yuri` syntax lowering for `if command == "help" { ... }`.

Empty

Writes the smallest source file: `export const name = "Tsundere"`. Use it when testing compiler behavior without Discord setup.

After Creating A Template

The CLI prints the next step as `cd <name> && tsundere install && tsundere dev`. That is the real expected flow. Install first so npm writes `package-lock.json` and Tsundere writes package snapshots. Then start dev mode so command discovery, build output, and runtime output are generated.

cd my-bot
tsundere install
tsundere dev

Common Template Mistakes

  • Expecting `tsundere create` to overwrite an existing folder. It will not.
  • Assuming REST and WebSocket templates install Express or ws for you. Run package install commands for dependencies you use.
  • Editing generated `.tsundere/runtime/discord` files instead of source or CLI runtime code.
  • Assuming disk files in `templates/` are the command source of truth. The current create command uses `src/cli.ts`.
