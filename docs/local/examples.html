Examples

The example `.yuri` files live in `docs/examples`. They are both learning material and compiler samples: this repository's `tsundere.config.json` points `source` at `docs/examples`, so `npm run build` compiles these examples through the current compiler.

The examples are not installed into new projects automatically. Copy the small pattern you need into your own `src` folder. Do not import directly from `docs/examples` in a production bot.

What The Examples Teach

Starter Bot

`starter-bot.yuri` shows `Client`, `Intents`, `Slash`, `Embed`, `Button`, `Row`, a `/ping` command, and a button interaction.

Embeds and Components

`embeds.yuri` and `components.yuri` show builder chaining, rows, buttons, select menus, and `parseCustomId`.

Modal and Slash Options

`modal-flow.yuri` builds modal payloads, while `slash-options.yuri` demonstrates `requireOption`, `resolveUser`, `resolveRole`, and `resolveChannel`.

Typed Router

`typed-router.yuri` uses `InteractionRouter`, `Component.define`, `Schema.define`, and `Select.define` for routed interaction handling.

Runtime Helpers

`cache-and-helpers.yuri`, `prefix-utils.yuri`, `rest-commands.yuri`, and `webhook-thread-audit.yuri` show runtime helpers outside the simplest slash command path.

Gateway and Sharding

`sharding-and-gateway.yuri` shows `Partials`, `shards: "auto"`, `client.shard(0)`, and gateway error handling.

Run The Examples Through The Compiler

From the repository root, the project config already points at `docs/examples`, with command discovery under `docs/examples/commands`. That means the normal build and command sync commands exercise the example set.

npm run build
node dist/cli.js commands sync
node dist/cli.js build

`npm run build` builds the CLI itself. `node dist/cli.js build` then uses the built CLI to compile the examples. If you are using an installed CLI instead of source, run `tsundere build` from the repository root after installing dependencies.

Command discovery reads only `docs/examples/commands` in this repository config. That means files such as `docs/examples/commands/ping.yuri`, `docs/examples/commands/avatar.yuri`, and `docs/examples/commands/admin/ban.yuri` are used for `.yuri-cache/discord.commands.json`. The top-level example files are still compiled, but they are not command-discovery entries unless the config changes.

Syntax Style In Examples

Some examples intentionally use Tsundere's current syntax lowering, such as `if interaction.isCommand("ping") { ... }`. That is valid for the current compiler, but the safest style for beginners remains close to JavaScript and TypeScript. If a copied example behaves strangely, simplify the shape before assuming the runtime is broken.

Several examples call real Discord login through `client.login()`. Do not run them against Discord until `DISCORD_TOKEN` is set and the bot intents match the example. For compiler-only learning, build the examples without starting them.

Examples that construct modals, components, or router definitions can be useful even if you do not run the full file. Copy the builder pattern, then place it inside your own command or interaction handler. For example, you might copy the `TextInput.paragraph(...).length(10, 1000)` pattern from `modal-flow.yuri` without copying the whole feedback command.

Compile-Only Versus Runtime Examples

A compile-only example is useful for checking the compiler, generated JavaScript, or docs pattern. A runtime example is something you actually start with Discord credentials. Most files in `docs/examples` import `@tsundere/discord` and call `client.login()`, so they become runtime examples if you execute the generated output.

Before running one, check three things: the token source, the intents, and the Discord permissions. `slash-options.yuri` needs guild/member context. `sharding-and-gateway.yuri` uses `GuildMembers` and `GuildVoiceStates`. Moderation command examples may need role hierarchy and permissions that a fresh bot invite does not have.

For safe experimentation, create a fresh project with the Discord template, copy one example concept into `src/main.yuri` or `src/commands`, run `tsundere install`, then use `tsundere dev`. That isolates the example from the repository's docs build config and makes the generated output easier to inspect.

Common Example Mistakes

  • Copying an entire example when you only need one small interaction pattern.
  • Running Discord examples without setting `DISCORD_TOKEN`.
  • Forgetting required intents such as `GuildMembers`, `GuildMessages`, `MessageContent`, or `GuildVoiceStates`.
  • Assuming examples are full applications with persistence, permissions, and deployment hardening.
  • Editing docs examples and forgetting that the repository build compiles them.
