Deployment

Tsundere deploys like a Node project: install dependencies, build `.yuri` files, provide environment variables, and start the generated runtime with `tsundere start`.

Overview

A deployed Tsundere app needs the CLI, Node.js, package dependencies, `.tsundere/runtime/discord`, generated runtime output, and production secrets. The local runtime means projects can resolve `@tsundere/discord` without depending on a public registry package.

Why It Exists

Discord bots often start as local scripts and become production services. A clear deployment path keeps the same project working on Windows, Linux, Docker, VPS hosts, CI runners, and future cloud function targets.

Basic Deployment Flow

npm install -g @tsundere/cli
tsundere runtime install
tsundere install
tsundere build
tsundere start

Use this flow on a server after cloning the project. In CI, run `tsundere doctor` before build when you want environment diagnostics.

Production Environment

DISCORD_TOKEN=...
DISCORD_GUILD_ID=...
COMMANDS_GLOBAL=true
LOGGING_ENABLED=true
NODE_ENV=production

For development, guild command sync is faster. For public production bots, global sync is usually expected, but command propagation may take longer.

Docker Shape

FROM node:22-slim
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npx tsundere runtime install && npx tsundere build
CMD ["npx", "tsundere", "start"]

This is a starter shape, not the final optimized image. Production images should pin versions, avoid copying local secrets, and use a non-root user when possible.

Systemd Shape

[Service]
WorkingDirectory=/srv/tsundere-bot
EnvironmentFile=/srv/tsundere-bot/.env
ExecStart=/usr/bin/tsundere start
Restart=always
RestartSec=5

Common Mistakes

  • Running `node build/main.ts` instead of `tsundere start`.
  • Forgetting `tsundere runtime install` when moving a project to a clean server.
  • Using guild command sync in production without setting the correct guild ID.
  • Committing `.env` or generated caches into the repository.
  • Expecting global slash commands to update instantly.

Performance Notes

Build once before starting. Avoid recompiling on every process boot in production. For large bots, shard intentionally, track gateway latency, and keep REST-heavy startup work behind rate-limit-aware queues.
