Compiler

The Tsundere compiler turns `.yuri` files into JavaScript or TypeScript that can run on Node.js while preserving source maps, module relationships, and Discord-aware diagnostics.

Overview

The current compiler is intentionally practical: it scans project files, lowers supported `.yuri` syntax into target output, writes normal build files, and creates optimized runtime output for `tsundere start`.

The long-term compiler direction is a full parser and typed AST pipeline with incremental builds, richer source maps, deeper Discord validation, and language server metadata.

Why It Exists

Tsundere needs to feel like its own ecosystem without breaking npm compatibility. The compiler is the bridge: developers write `.yuri`, but the final runtime still behaves like a Node project.

When To Use It

  • Use `tsundere build` before production deploys.
  • Use `tsundere dev` when you want watch mode and runtime restarts.
  • Use source maps when debugging generated output back to `.yuri` lines.
  • Use diagnostics to catch Discord intent, permission, and builder mistakes before runtime.

Build Flow

.yuri files
  -> project scanner
  -> syntax lowering
  -> diagnostics
  -> build/*.js or build/*.ts
  -> .tsundere/runtime-build/*.js
  -> tsundere start

Configuration Example

{
  "entry": "src/main.yuri",
  "outDir": "build",
  "runtimeOutDir": ".tsundere/runtime-build",
  "target": "javascript",
  "sourceMaps": true,
  "strict": true,
  "diagnostics": {
    "warnings": true,
    "verbose": false,
    "color": true
  }
}

Keep source maps on during development. For protected production builds, source maps should be stripped unless you have a private symbol server or internal artifact store.

Advanced Example

{
  "commands": {
    "discovery": true,
    "routeBased": false,
    "directory": "./src/commands",
    "groups": {
      "./src/commands/admin": {
        "routeBased": true,
        "groupName": "admin"
      }
    }
  }
}

Command discovery is compiler-adjacent because the project scanner can understand files, routes, exports, and command metadata before the bot starts.

Common Mistakes

  • Do not edit generated files in `build` or `.tsundere/runtime-build`; edit source `.yuri` files.
  • Do not rely on global command sync timing while testing command changes.
  • Do not suppress warnings permanently when they point to missing intents or unsafe Discord operations.
  • Do not assume every future `.yuri` syntax idea already exists in the current compiler.

Performance Notes

Large-project performance should come from incremental file scanning, AST caching, dependency graph caching, and parallel compilation. Today, keep projects organized and avoid unnecessary generated files in source directories.
