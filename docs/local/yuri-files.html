Getting Started

.yuri Files

`.yuri` files are the source files Tsundere compiles. They are intentionally close to JavaScript and TypeScript today, with extra lowering for Discord-oriented blocks and a few friendlier syntax forms.

The Safest Style

The current compiler in `src/compiler/transpile.ts` is not a full AST parser. It performs ordered transforms and diagnostics. The most reliable code looks like normal TypeScript-style JavaScript with imports, `const`, functions, classes/interfaces, object literals, and calls to `@tsundere/discord` helpers.

import { Client, Intents, Slash, Embed } from "@tsundere/discord"

const client = new Client({
  token: env.DISCORD_TOKEN,
  intents: [Intents.Guilds, Intents.GuildMessages]
})

client.once("ready", () => {
  log(`Online as ${client.user.tag}`)
})

When the compiler sees `env.` it adds `const env = process.env`. When it sees `log(` or `print(` it adds a console helper. Imports from `tsundere/discord` are normalized to `@tsundere/discord`.

Think of this as a practical source format rather than a finished new programming language. The compiler walks every `.yuri` file under the configured `source` directory, applies text-aware transforms, emits `.ts` or `.js` into `outDir`, and then emits runnable JavaScript under `.tsundere/runtime-build`. If a feature looks like normal JavaScript or TypeScript, it is usually the safest choice.

Local imports should be written without generated-output paths. For example, import `./services/users` from source. During runtime emission, Tsundere adds `.js` extensions to relative ESM imports so Node can resolve the generated files. Do not import from `.tsundere/runtime-build` inside source files.

Lowered Syntax

Tsundere currently lowers several forms: TypeScript-like functions with `fn`, simpler `if condition {}` forms, Spanish-inspired keywords such as `sea`, `retorna`, `espera`, `si`, and `sino`, and Discord blocks such as `embed`, `button`, `row`, `client`, `bot`, and `command`. These are useful, but because they are implemented through transforms, keep nesting simple and test output with `tsundere build`.

embed {
  title: "Pong"
  description: `Latency: ${client.ping}ms`
  color: "#ff7ab6"
}

That style is lowered into chained builder calls. If a transform does not understand the shape you wrote, use normal JavaScript-like calls instead.

The compiler also recognizes `client name { ... }` and `bot name { ... }` blocks that create a `Client`, event blocks such as `on ready { ... }`, and object-call blocks like `interaction.reply { content: "pong" }`. These are conveniences. They are not a promise that every nested block or future syntax idea is complete today.

When documenting or teaching Tsundere code, prefer examples that compile through the current implementation. The beginner-safe form is still explicit imports plus normal calls: `Client`, `Slash.command`, `Embed.create`, `Button.secondary`, `Row.of`, and `client.on`. That style is easier to compare against the generated JavaScript when troubleshooting.

What Build Output Looks Like

If `target` is `typescript`, `src/main.yuri` first becomes `build/main.ts`. The runtime emitter then uses TypeScript's `transpileModule` to turn that file into `.tsundere/runtime-build/main.js`. If `target` is `javascript`, Tsundere writes JavaScript earlier and copies it into runtime output after import extension cleanup. Source maps are added when `sourceMaps` is true.

Open the generated file only to understand what happened. The generated file is not where you fix bugs. A clean edit loop is: change `.yuri`, run `tsundere build`, read diagnostics, and then inspect output if the emitted JavaScript is surprising. This keeps source, build output, and runtime output from drifting apart.

Diagnostics You May See

  • `YURI101`: strict mode warns when it sees `any`.
  • `YURI210`: pattern matching is recognized but not fully lowered yet.
  • `DISCORD002` and `DISCORD003`: event code may need Discord intents such as `GuildMessages`, `GuildMembers`, or `MessageContent`.
  • `DISCORD005`: embed title or description exceeds Discord limits.
  • `DISCORD010`: component custom IDs exceed Discord's 100 character limit.
  • `DISCORD020`: code appears to require Discord permissions such as `BanMembers`, `ManageRoles`, or `ViewAuditLog`.

Diagnostics can be filtered by config. `diagnostics.warnings: false` suppresses warnings, and `diagnostics.disabled` can suppress specific diagnostic codes. Errors still stop the build. Keep warnings on while learning; they are useful because they catch Discord intent and permission mistakes before you stare at a silent bot.

Common Mistakes

Do not assume every future Yuri idea exists today. If the compiler emits surprising JavaScript, simplify to normal TypeScript-like syntax. Also remember that `.yuri` output is ESM; imports must resolve in Node after Tsundere adds `.js` extensions for local imports in runtime output.
