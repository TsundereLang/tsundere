Discord Collectors

Collectors wait for a limited set of interactions or messages and then stop by time, count, or custom logic.

Overview

Collectors are useful for confirmations, polls, setup flows, modal follow-ups, and temporary component workflows. Tsundere should make timeout handling and typed interaction responses obvious.

Why It Exists

Without collectors, bots often create permanent listeners for temporary flows. Collectors constrain the lifetime of a workflow and reduce accidental duplicate handlers.

When To Use It

  • Use collectors for temporary decision flows.
  • Use routers for long-lived global component handlers.
  • Use timeouts to clean up stale UI.
  • Use filters to restrict users and components.

Basic Example

const collector = Collectors.component({
  message,
  time: 30_000,
  filter: (i) => i.user.id == interaction.user.id
})

collector.on("collect", async (button) => {
  await button.update({ content: "Confirmed", components: [] })
})

Advanced Example

collector.on("end", async (items, reason) => {
  if reason == "time" {
    await interaction.editReply({
      content: "This action expired.",
      components: []
    })
  }
})

Common Mistakes

  • Forgetting to remove or disable components after timeout.
  • Allowing anyone to click a private confirmation button.
  • Creating global listeners for one-off flows.
  • Not handling collector end reasons.

Performance Notes

Collectors should be bounded. Long-lived collectors with broad filters can behave like accidental memory leaks.
