Overview

Empty Project is a practical starter example based on the `empty` style. It explains the project shape, how to run it, and what to change before using it in production.

Purpose And Use Cases

Use this example as a starting point, not as a locked-in framework. Copy the structure, replace placeholders, and keep project-specific behavior inside your own services.

Architecture Or Design Notes

Example architecture: src/main.yuri, tsundere.config.json, package.json. Start from the template, keep `main.yuri` focused on startup, and move reusable logic into services so the example can grow without becoming tangled.

Beginner Example

tsundere create scratch --template empty
cd scratch
tsundere dev

Advanced Example

Add one service at a time and keep generated output out of source control.

Reference And API Details

Reference layout: src/main.yuri, tsundere.config.json, package.json. Template: `empty`. Production notes should cover secrets, command sync scope, generated runtime folders, and where project-specific services belong.

Best Practices

  • Treat examples as starting points, not production policy.
  • Replace placeholder IDs, tokens, channels, and guild settings immediately.
  • Move real business logic into services so handlers stay readable.
  • Add tests before turning an example into a deployed bot.

Common Mistakes

  • Leaving example command names, guild IDs, or channel IDs unchanged.
  • Copying example structure but skipping `.env.example` updates.
  • Turning examples into a hidden framework with too much shared state.
  • Deploying an example before checking permissions and command sync scope.

Troubleshooting

  • Run the example exactly once before customizing it.
  • Check `.env.example` and replace placeholders with real local values.
  • Use `tsundere build` before `tsundere dev` if syntax errors are unclear.
  • When Discord commands do not appear, confirm guild sync settings and restart the bot.
