Overview

dev is a Tsundere CLI command. Compile, run, watch, and restart a .yuri project during local development. The command is designed for normal project folders, CI jobs, and local development terminals.

Purpose And Use Cases

Start it from the project root after `.env` and `tsundere.config.json` are ready. It watches source files and restarts the runtime build when they change. This matters because Tsundere is meant to feel like a normal developer runtime: commands should work in any project folder, produce readable output, and leave source code separate from generated runtime files.

Architecture Or Design Notes

Command architecture: input is read from the current working directory, configuration is resolved from `tsundere.config.json`, package/runtime state is kept in normal project files plus `.tsundere`, and output is designed to be scriptable. Syntax: `tsundere dev`.

Beginner Example

tsundere dev
# edit src/main.yuri
# the runtime rebuilds and restarts

Advanced Example

tsundere dev
# edit src/main.yuri
# the runtime rebuilds and restarts

# Production check
tsundere build && npm test

Reference And API Details

Syntax: `tsundere dev`. Common flags: --no-sync, --warnings=false, --verbose. Typical workflow: Start it from the project root after `.env` and `tsundere.config.json` are ready. It watches source files and restarts the runtime build when they change. Related files include `package.json`, `tsundere.config.json`, `tsundere-lock.yaml`, `tsundere-workspace.yaml`, `.tsundere/runtime-build`, and `build` depending on the command.

Best Practices

  • Run `tsundere dev` from the project root unless the command says otherwise.
  • Keep generated output out of source edits.
  • Use command output in CI so failures are visible in logs.
  • Pair CLI changes with `tsundere doctor`, build, and tests before release.

Common Mistakes

  • Running the command from the wrong folder.
  • Ignoring a failed install or build before starting the runtime.
  • Confusing `tsundere update ` with `tsundere updater` for toolchain updates.
  • Skipping the command-specific troubleshooting note: If the process does not restart, confirm the source directory in `tsundere.config.json` points at the folder you are editing.

Troubleshooting

  • If the process does not restart, confirm the source directory in `tsundere.config.json` points at the folder you are editing.
  • Run `tsundere doctor` for environment and runtime checks.
  • Run with verbose diagnostics when command output is too short.
  • Check project config, package metadata, and generated runtime directories.
