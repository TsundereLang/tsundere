Discord Embeds

Embeds create readable, structured Discord messages for status, results, logs, and dashboards.

Overview

Tsundere's embed builder keeps chaining familiar while making validation and autocomplete more obvious. It should warn about title, description, field, and total text limits before runtime.

Why It Exists

Embeds are easy to overfill. A professional wrapper should help developers create clean messages and catch Discord API limit errors before deployment.

When To Use It

  • Use embeds for command output with multiple fields.
  • Use plain content for one-line replies.
  • Use consistent colors for success, warning, and error states.
  • Use timestamps for logs and audit trails.

Basic Example

const card = Embed.create()
  .title("Pong")
  .description(`Latency: ${client.ping}ms`)
  .color("#ff7ab6")
  .timestamp()
  .toJSON()

Advanced Example

const report = Embed.create()
  .title("Moderation Action")
  .field("User", target.tag, true)
  .field("Moderator", interaction.user.tag, true)
  .field("Reason", reason)
  .footer("Tsundere Audit")
  .color(theme.warning)
  .timestamp()

Common Mistakes

  • Sending builder objects without `.toJSON()` when the runtime expects JSON.
  • Exceeding Discord's 25-field limit.
  • Using unreadable color semantics.
  • Putting secrets or private moderation notes in public embeds.

Performance Notes

Embed creation is cheap. The expensive part is usually fetching data to populate embeds. Batch database reads and avoid per-field API calls.
