Roadmap

This roadmap describes what exists in this repository today and what is still planned. It is not a sales page. Tsundere already has a working CLI, `.yuri` transpilation, Discord wrapper package, command discovery, package optimizer, updater, Linux and Windows platform helpers, local docs, VS Code extension support, protected runtime builds, and distributed runtime foundations. It does not yet have a full parser, a real plugin marketplace, a command graph UI, or production-grade multi-machine orchestration.

Use this page to understand direction before contributing. If a feature is listed as shipped, there is source code or a CLI command behind it. If it is listed as experimental, it works in a limited form and needs hardening. If it is listed as planned, do not build production workflows around it yet.

What This Page Teaches

  • Which roadmap items are implemented in the current codebase.
  • Which features are foundations rather than finished product surfaces.
  • Which commands you can run now to verify a feature.
  • Where the relevant source lives for each work area.
  • Which old roadmap claims should be treated as future ideas, not current behavior.

What Is Shipped Now

The shipped parts are the features you can verify from a checkout after `npm install` and `npm run build`. These are not necessarily final, but they are real code paths with tests or CLI commands behind them.

Project Lifecycle

`tsundere create`, `install`, `dev`, `build`, `start`, `format`, `lint`, `test`, `docs`, and `doctor` are routed through `src/cli.ts`. Generated projects include `package.json`, `tsundere.config.json`, `.env.example`, `.gitignore`, `src/main.yuri`, and, for Discord templates, `src/commands/ping.yuri`.

Discord Runtime

The bundled wrapper exposes client helpers, intents, embeds, slash command builders, components, interactions, collectors, REST helpers, prefix helpers, and a native runtime bridge. The local project runtime is refreshed into `.tsundere/runtime/discord` during builds and installs.

Package Optimizer

Tsundere wraps npm rather than replacing it. The optimizer reads `package-lock.json`, harvests package directories into a store, validates directory hashes before reuse, hydrates with hard links or copies, and writes `tsundere-workspace.yaml` plus `tsundere-lock.yaml` after successful npm operations.

Local Docs and Tests

Local docs live under `docs/local`. The repo now includes `npm run docs:check`, unit tests under `tests`, runtime stress tests, and system stress tests that exercise compiler, package, platform, updater, and distributed runtime surfaces.

Experimental Foundations

Experimental means the feature exists, but the page should not make it sound complete. These items are useful for local projects and testing, but they need more design, stronger parsing, broader tests, or better packaging before they become stable promises.

Tsundere Protect

`protectJavaScript` strips comments and source map comments, encodes static strings, adds a build ID, injects guard code for advanced and maximum profiles, and emits `.tsundere/runtime-build/tsundere-protect.json`. It is protection against casual inspection, not a security boundary.

Read Protect docs

Discord Intelligence

The compiler already warns about some intents, message content, permissions, embed limits, and custom ID limits. The VS Code extension contributes Discord completions, hovers, snippets, and type sync. A unified intelligence dashboard is still future work.

Read Intelligence docs

Distributed Runtime

Runtime planning, local IPC, global events, memory cache, singleton tasks, Prometheus metrics, Grafana export, inspect output, and stress reports exist. Real multi-process or multi-machine orchestration is not finished.

Read CLI runtime docs

Type Bridge

`tsundere types sync` extracts Discord type metadata and writes JSON files in `.yuri-cache`. The VS Code extension can trigger sync when package files change. This is useful, but not yet a complete language server type system.

Read Discord docs

Planned Work

These items are on the roadmap because the codebase has the first pieces or a clear reason to need them, but they should not be documented as available commands unless the implementation lands.

A Real Parser and AST Pipeline

`src/compiler/transpile.ts` is still regex and block-transform heavy. A full parser would make Yuri syntax more predictable, improve diagnostics, support better formatting, and reduce surprising lowering edge cases.

Command Visualizer

Command discovery already writes route metadata. The planned visualizer should use that data, plus component and modal metadata, to show command graphs and duplicate or stale command warnings in the editor. There is no VS Code graph panel yet.

Read Visualizer roadmap

Plugin Marketplace

`src/plugins.ts` defines plugin shapes for compiler, language server, lint, CLI, and generators. The CLI can map plugin names to npm package names or accept package specs. A searchable registry, marketplace UI, and loaded plugin execution pipeline are not complete.

Read Plugin roadmap

Release Trust and Installer Hardening

The updater can download and install CLI tarballs, and release scripts can assemble artifacts. Audit notes still call out checksum or signature verification, stronger path guards in installer scripts, and safer release deletion behavior as remaining work.

Commands That Prove Current Behavior

Run these from a source checkout after dependencies are installed. They verify the current foundations without requiring Discord credentials.

npm run build
npm run test:unit
npm run docs:check
npm run stress:runtime
npm run stress:system
node dist/cli.js doctor
node dist/cli.js inspect
node dist/cli.js metrics doctor
node dist/cli.js updater self --dry-run

`updater self --dry-run` still needs network access and a release with a valid CLI tarball. If it cannot read GitHub from your environment, the command should fail cleanly with a network or release-repo message.

Common Roadmap Misreads

Protect Is Not DRM

Protected output raises effort for casual inspection and adds traceable fingerprints. It cannot make JavaScript impossible to reverse engineer.

Plugins Are Not Fully Loaded Yet

The interface and install command exist, but the compiler does not yet execute arbitrary plugin hooks across a marketplace registry.

Stress Tests Are Local

`tsundere stress runtime` and `tsundere stress system` exercise local runtime foundations and fixtures. They do not connect to Discord or prove real gateway scaling.

Yuri Is Still Close to JS

The safest production code stays near TypeScript and JavaScript shapes until the parser and language server mature.

How to Suggest Roadmap Work

Good roadmap requests include a real bot workflow, the command you wish existed, the source files it would touch, and what failure it would prevent. For Discord problems, include intents, permissions, package versions, and a small `.yuri` sample. For compiler work, include the current emitted output and the output you expected.
