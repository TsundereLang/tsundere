Getting Started

Windows Setup

Windows support is first-class in the CLI path handling. npm and pnpm are invoked through `.cmd` shims, local Tsundere storage is under your home directory, and the release installer is PowerShell-based.

Prerequisites

Install Node.js with npm enabled. Then open a new PowerShell window and confirm both commands are available:

node --version
npm --version

The platform layer maps `npm`, `pnpm`, and `npx` to `.cmd` on Windows. Command lookup uses `where.exe`. Opening docs uses `cmd.exe /c start`.

Use the same terminal family for install and project work. If you install with Windows PowerShell, create and run projects from Windows paths such as `C:\Users\you\Projects\my-bot`. If you install inside WSL, follow the Linux page and keep the project inside the Linux filesystem. Mixing Windows global npm shims with WSL paths is one of the fastest ways to get confusing resolution errors.

PowerShell Installer

The script `scripts/install-tsundere-windows.ps1` fetches the latest release metadata, downloads a zip, extracts it to a temporary folder, finds `tsundere-cli.tgz`, checks Node and npm, installs pnpm if missing, installs the CLI globally, optionally installs the editor extension, runs `tsundere help`, and cleans up the temporary folder.

powershell -ExecutionPolicy Bypass -File scripts/install-tsundere-windows.ps1

If PowerShell blocks the script, check execution policy and whether the file came from the internet. If the script cannot find `tsundere-cli.tgz`, the release bundle is missing the CLI package.

The script downloads a zip asset from the latest GitHub release when one exists. If the release has no zip asset, it falls back to GitHub's source zipball. That fallback can download source code successfully and still fail later if the release did not include the packaged CLI tarball. For installer issues, always check the release assets before changing local project files.

The script also installs pnpm globally if it is missing. Tsundere project installs are npm-first, but the repository and some packaging flows use pnpm-style commands or compatibility fallbacks. If your machine blocks global npm installs, fix npm global permissions or use a Node version manager before rerunning the installer.

Storage Paths

The platform module uses a `.tsundere` folder inside your Windows home directory:

%USERPROFILE%\.tsundere\
%USERPROFILE%\.tsundere\config.json
%USERPROFILE%\.tsundere\cache\
%USERPROFILE%\.tsundere\store\
%USERPROFILE%\.tsundere\logs\

Project-generated files still live inside the project: `.tsundere/runtime/discord`, `.tsundere/runtime-build`, `.yuri-cache`, `build`, and npm files.

The global store path can be changed with `storePath` in `tsundere.config.json`, but keep it on a local filesystem when possible. Hard links work only when the store and project are on compatible volumes. If hard links fail, `linkMode: "auto"` falls back to copying. Use `linkMode: "copy"` when you are debugging network drives, removable drives, or security software that blocks link creation.

Running A Project On Windows

After installation, the normal project loop is the same as every other platform:

tsundere create my-bot --template discord
cd my-bot
tsundere install
tsundere dev

If dev mode restarts too often or misses changes, move the project out of aggressively synced folders and try again. The dev watcher has a polling fallback, but the cleanest setup is still a normal local project folder with stable file modification times.

Windows-Specific Problems

  • If `tsundere` is not recognized after global install, close and reopen the terminal so PATH refreshes.
  • If you work inside OneDrive, dev mode may rely on polling more often because recursive watch behavior can be inconsistent.
  • If WSL is involved, prefer the Linux installer inside WSL or the Windows installer from a Windows terminal. Mixing Windows npm with WSL UNC paths is fragile.

For long paths, use a short project root such as `C:\Projects\my-bot`. The code uses `path.resolve`, `path.join`, and Windows-specific path helpers, but Discord bots often add logs, generated files, and nested packages. Short roots make npm and editor tooling less brittle.
