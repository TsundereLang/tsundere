Discord Components

Components are buttons, select menus, rows, and newer layout elements that let users interact without typing commands.

Overview

Tsundere exposes builder helpers for buttons, selects, rows, and Discord's newer component layouts. The goal is clean construction, typed custom IDs, editor completion, and compile-time Discord limit warnings.

Why It Exists

Raw component IDs often become fragile strings like `ban_123`. Tsundere's component direction makes encoded data typed and keeps handlers from hand-parsing IDs.

When To Use It

  • Use buttons for short decisions.
  • Use selects when users choose from many options.
  • Use rows to group related controls.
  • Use layout components for richer message surfaces when supported by Discord clients.

Button Example

import { Button, Row } from "@tsundere/discord"

const actions = Row.of(
  Button.success("confirm_purchase").label("Confirm"),
  Button.danger("cancel_purchase").label("Cancel")
)

Typed Component Example

const BanButton = Component.define<{
  userId: string
}>("ban")

const button = Button.danger(
  BanButton.create({ userId: member.id })
).label("Ban")

Advanced Layout Example

const panel = Components.container()
  .accentColor("#ff7ab6")
  .add(
    Components.section()
      .content("Release notifications")
      .accessory(Button.primary("role_release").label("Toggle"))
  )

Common Mistakes

  • Putting too many buttons in one row.
  • Using custom IDs longer than Discord allows.
  • Assuming a button interaction has command fields.
  • Forgetting to handle timeout or stale component state.

Performance Notes

Component handlers should respond quickly. Defer updates for longer work and avoid database calls before acknowledging the interaction.
