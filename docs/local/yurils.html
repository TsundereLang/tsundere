YuriLS

YuriLS is the Tsundere language server. It powers editor completions, hover docs, diagnostics, go-to-definition, and Discord-aware intelligence for `.yuri` files.

Overview

The language server indexes the workspace, parses open files, tracks imports and symbols, reads generated Discord metadata, and returns language server protocol responses to editors.

Why It Exists

A custom language only feels real when the editor understands it. YuriLS makes `.yuri` feel intentional instead of being plain text with a custom extension.

When To Use It

  • Use YuriLS through the official VS Code or Cursor extension.
  • Restart YuriLS after changing generated type metadata.
  • Run `tsundere types sync` when Discord packages change.
  • Use diagnostics to catch syntax, import, config, and Discord usage problems.

Discord Metadata Flow

discord.js .d.ts files
  -> TypeScript extractor
  -> intermediate type graph
  -> Tsundere type mapper
  -> .yuri-cache/discord.*.json
  -> YuriLS completions and hovers

Advanced Example

if (interaction.isButton()) {
  interaction.customId
  // YuriLS should narrow interaction to ButtonInteraction here.
}

Type narrowing is a major editor goal. Checks like `isCommand`, `isButton`, `isModalSubmit`, and select menu guards should influence completions inside the block.

Common Mistakes

  • Do not expect Discord completions if `@tsundere/discord` is not installed or metadata is missing.
  • Do not keep stale `.yuri-cache` files after major Discord package updates.
  • Do not confuse TypeScript signatures with Tsundere display signatures; YuriLS should show `.yuri`-style signatures.
  • Do not ignore import diagnostics; missing local runtime packages usually break Node startup too.

Performance Notes

Language servers must stay responsive. YuriLS should cache parsed files, debounce diagnostics, hash metadata inputs, and avoid regenerating Discord types on every keystroke.
