Getting Started

Installation Troubleshooting

Most Tsundere setup problems come from four places: Node/npm is not available, the release bundle is incomplete, the local Discord runtime package is missing, or platform permissions/path lookup are failing.

Start With Doctor

tsundere doctor

Doctor prints the project name, source, output, target, strict mode, platform, Tsundere home/config/cache/log paths, Node/npm availability, Linux executable checks when relevant, and package optimizer store status. If npm is missing, install or repair Node before debugging Tsundere.

Run doctor from the project root when debugging a project. The command loads the config in the current directory, so running it from your home folder can show default config instead of the bot you are trying to fix. If doctor says `source: src` but your project uses a different folder, either change directory or update `tsundere.config.json`.

`@tsundere/discord` Cannot Resolve

New projects depend on `@tsundere/discord` through `file:.tsundere/runtime/discord`. If that local package folder is missing or stale, Node cannot resolve the import.

tsundere runtime install
tsundere install

`runtime install` forces the local runtime package manifest into place. `install` lets npm install the file dependency and lets Tsundere repair optimizer metadata.

If this still fails, open `package.json` and confirm the dependency points to `file:.tsundere/runtime/discord`. Then check that `.tsundere/runtime/discord/package.json` exists and has `"name": "@tsundere/discord"`. If you are working from a source checkout of the CLI, run `npm run build` in the CLI repository so `dist/discord` exists before trying to create or repair projects.

Release Installer Cannot Find `tsundere-cli.tgz`

Both web installer scripts search the downloaded release contents for `tsundere-cli.tgz`. If the latest release only contains source code or missing assets, the installer cannot complete. Use a release built by `npm run dist:release`, or install from source while the release is repaired.

This is a release packaging problem, not a local bot problem. The installer has already reached GitHub and downloaded something; it just did not find the asset layout it expects. Check the GitHub release attached files before changing project code. If you are preparing a release, make sure the generated tarball is attached to the release that the installer resolves as latest.

`tsundere` Command Not Found

This usually means npm installed the package somewhere that is not on PATH, or the terminal was opened before the PATH changed. Reopen the terminal first. Then check where npm puts global binaries.

npm prefix -g
npm bin -g

On Windows, the command may resolve through a `.cmd` shim. On Linux and macOS it should be an executable file. If you installed from source but only ran `node dist/cli.js`, you did not install a global `tsundere` command; either use the node command while developing or install/link the package intentionally.

Cache or Store Looks Broken

tsundere store path
tsundere store prune
tsundere cache clean

The optimizer validates stored package metadata and directory hashes before reuse. `store prune` removes unreferenced or invalid entries inside the configured store. `cache clean` removes the store and recreates it. Neither command should delete project source files.

Use `store prune` first when the store is merely stale. Use `cache clean` when you want to force a full rebuild of the Tsundere store. After cleaning, the next `tsundere install` may be slower because packages have to be harvested again from npm-installed `node_modules`.

Windows and Linux Notes

On Windows, npm and pnpm commands resolve through `.cmd` shims. On Linux and macOS, command lookup uses `sh -c command -v`. On Linux, doctor also checks executable permissions for scripts such as `scripts/install-linux.sh` and `dist/cli.js` when they exist.

For Windows, avoid running projects from protected locations and be careful with OneDrive paths if file watching behaves strangely. For Linux, `permission denied` usually means the script exists but is not executable, or the directory is mounted with restrictive options. Doctor tries to apply executable bits where it can, and Linux-specific errors include chmod guidance.

When To Reinstall Dependencies

Re-run `tsundere install` after editing dependencies, switching branches, deleting `.tsundere/runtime/discord`, cleaning the Tsundere store, or pulling changes that alter package files. Re-run plain `npm install` only when you intentionally want to bypass Tsundere's optimizer layer for diagnosis. If plain npm works but `tsundere install` does not, the issue is likely in the optimizer or local runtime repair step.
