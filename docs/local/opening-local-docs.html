Getting Started

Opening Local Docs

Tsundere ships local HTML docs with the CLI package. They do not require a documentation server; the CLI opens the installed `docs/local/index.html` file in your system browser.

Open the Docs

tsundere docs

The CLI resolves `docs/local/index.html` relative to the installed CLI root. If it exists, Tsundere prints the path and calls the platform opener: `cmd.exe /c start` on Windows, `open` on macOS, and `xdg-open` on Linux.

No dev server is required for normal reading. These pages are static HTML files with shared CSS and JavaScript. The tradeoff is that browser security rules and relative paths matter: open the HTML file from its real `docs/local` folder so `theme.css`, `theme.js`, and the logo path resolve correctly.

Reading Docs From Source

In a source checkout, open this file directly:

docs/local/index.html

The pages load `theme.css` and `theme.js`, and many pages reference `../../assets/tsundere-logo.png`. If you move an HTML page outside `docs/local`, those paths will break.

From the repository, the docs are also covered by `npm run docs:check`. That script verifies required Getting Started pages, checks local references in every HTML page, and confirms the Getting Started pages are present in the shared theme index. It is a lightweight docs build gate for static files.

npm run docs:check

Because the docs are static, editing them is straightforward: change the HTML file, update `theme.js` if navigation or search needs to know about it, then run the checker. There is no markdown compilation step right now. That makes docs easy to ship in a release bundle, but it also means shared layout changes must be kept consistent manually.

Installed Package Versus Repository Docs

When you run `tsundere docs` from an installed CLI, you are opening the docs that came with that installed package. When you open `docs/local/index.html` from a repository checkout, you are reading the docs in your working tree. If you are testing documentation changes, use the repository file. If you are helping a user understand their installed version, ask them what `tsundere version` prints and remember their local docs may be older than your branch.

This distinction matters after releases. A docs fix committed to the repo does not automatically update every installed CLI. It must be included in a release package, then installed or updated on the user's machine. If a user says the docs are wrong, ask whether they are reading GitHub, a source checkout, or the installed local docs.

Search, Theme, and Navigation

The search box is generated by `docs/local/theme.js`. That script owns the docs search index and dynamically adds a few navigation links and dropdowns. When new local docs pages are added, the search index and dropdowns must be updated manually.

The theme script also adds the author strip, dark/light toggle, Getting Started dropdown, Discord dropdown, Roadmap dropdown, and search result snippets. If a page exists but never appears in search, the page file alone is not enough; add an entry to `docsIndex` with text that includes the terms users are likely to search.

Because navigation is partly generated at runtime, view a changed page in a browser after editing `theme.js`. A syntax error in that shared script can break search and navigation across every docs page even though the HTML files still open.

The sidebar HTML inside each page still includes base links. `theme.js` enhances that base navigation after the page loads. This is why docs pages can look simple in source but show dropdowns in the browser. When adding a whole new section, update both the base nav pattern in the HTML pages you touch and the generated dropdown/search behavior when needed.

Troubleshooting

  • If `tsundere docs` says local docs were not found, the installed package is missing the `docs` directory.
  • If Linux cannot open the browser, install an `xdg-open` provider or open the HTML file manually.
  • If search does not find a new page, update `docs/local/theme.js`.

If the docs open but styling is missing, check that you are opening the file from the real docs folder and that `theme.css` sits next to the HTML file. If the logo is missing, check `assets/tsundere-logo.png` relative to the repository root or installed package root.
