Discord Interactions

Interactions are Discord's structured user actions: slash commands, buttons, select menus, modals, autocomplete, and context menus.

Overview

Tsundere keeps `interactionCreate` familiar while adding type narrowing, helper methods, hover docs, response timing guidance, and diagnostics for common Discord API mistakes.

Purpose And Use Cases

Use interactions for command execution, component clicks, modal submissions, select choices, autocomplete responses, and context menu actions. They provide a cleaner user experience than parsing raw messages.

Architecture Notes

Large bots should route interactions by type and identifier. Keep the raw event listener small, then delegate command, button, modal, and select logic into focused handlers.

Beginner Example

client.on("interactionCreate", async (interaction) => {
  if (interaction.isCommand("ping")) {
    await interaction.reply({
      content: "pong",
      ephemeral: true
    })
  }
})

Advanced Example

client.on("interactionCreate", async (interaction) => {
  if (await commandRouter.handle(interaction)) return
  if (await componentRouter.handle(interaction)) return
  if (await modalRouter.handle(interaction)) return
  if (await selectRouter.handle(interaction)) return
})

Reference And API Details

Important interaction helpers include `reply`, `deferReply`, `editReply`, `followUp`, `deleteReply`, `isCommand`, `isButton`, `isModal`, `isSelectMenu`, `user`, `guild`, `channel`, `member`, `commandName`, `options`, and `customId`.

Best Practices

  • Acknowledge interactions quickly.
  • Use `deferReply` for work that may take more than a moment.
  • Use type guards before reading command, button, modal, or select-specific fields.
  • Use typed custom IDs for component data.

Common Mistakes

  • Replying twice to the same interaction.
  • Waiting too long before replying or deferring.
  • Reading `customId` on a command interaction.
  • Using ephemeral response options in unsupported response flows.

Troubleshooting

  • If commands do not appear, confirm command sync route and guild/global settings.
  • If a button does nothing, check the custom ID and handler filter.
  • If a reply fails, check whether the interaction was already acknowledged.
  • If autocomplete is stale, regenerate Discord type metadata and restart YuriLS.

Performance Notes

Keep interaction handlers thin. Defer early, move database calls into services, and measure command latency with `@tsundere/metrics`.
