VS Code Extension

The official extension makes `.yuri` files feel native in VS Code and Cursor with syntax highlighting, file icons, commands, diagnostics, and YuriLS integration.

Overview

The extension packages editor support, starts YuriLS, watches project files, surfaces diagnostics, and provides commands for common Tsundere workflows.

Why It Exists

Discord development is editor-heavy. A good extension should autocomplete events, builder methods, command options, interaction methods, and imports while explaining required intents and permissions in hover docs.

When To Use It

  • Install it when editing `.yuri` files in VS Code or Cursor.
  • Use it to run `Tsundere: Build Project` and `Tsundere: Run Dev Server` from the command palette.
  • Use it after `tsundere types sync` to inspect generated Discord IntelliSense.
  • Use it when onboarding new developers so docs, examples, and commands are discoverable.

Completion lists should include docs, parameter types, return types, and examples, not just method names.

Common Mistakes

  • Do not install an old VSIX after rebuilding a newer extension.
  • Do not forget to restart YuriLS when generated metadata changes.
  • Do not expect completions in files that VS Code does not recognize as `.yuri`.
  • Do not confuse extension install with project dependency install; both matter.

Performance Notes

The extension should avoid constant type regeneration. It should hash package versions, lockfiles, compiler versions, and metadata inputs, then regenerate only when something meaningful changes.
