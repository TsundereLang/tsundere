Discord Modals

Modals collect structured text input from users after a command or component interaction.

Overview

Tsundere modal helpers focus on schema-shaped data, typed submissions, clean custom IDs, and validation. Modals are best for short forms, reports, configuration, and confirmation details.

Why It Exists

Discord modal data arrives as component fields. Tsundere should make those fields feel typed and validated instead of requiring manual lookups everywhere.

When To Use It

  • Use modals for multi-field text input.
  • Use slash options for simple required values.
  • Use select menus for predefined choices.
  • Use validation before saving user input.

Basic Example

const reportModal = Modal.create("report", "Submit Report")
  .row(TextInput.paragraph("reason", "Reason").required())
  .row(TextInput.short("proof", "Proof").required(false))

Advanced Schema Direction

const ReportModal = Modal.define({
  reason: String,
  proof: String?
})

client.modal(ReportModal, async (ctx) => {
  await saveReport(ctx.data.reason, ctx.data.proof)
})

Common Mistakes

  • Adding too many inputs.
  • Not validating max length.
  • Assuming modal submissions always come from the original user.
  • Storing sensitive reports in public logs.

Performance Notes

Acknowledge modal submissions quickly. Push long writes to services and show a short success or failure reply.
